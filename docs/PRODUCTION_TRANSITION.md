# PRODUCTION_TRANSITION — 운영 전환 계획

MVP(범위 §CLAUDE.md 4)에서 제외했던 운영 항목을 정리한다. 본 사이클에서는 **2·3·5를 구현**하고,
**4·1·6은 사내 인프라 사양 의존이 커서 문서화만 하고 추후 작업**한다.

---

## 추후 작업 (문서화만, 이번엔 보류)

### 4. 운영 인프라·안정성 [보류]
- 시크릿 매니저로 `.env` 키 이전(사내 볼트), Qdrant 영속 스토리지·백업·모니터링.
- 헬스체크/알람, 구조적 로깅, 동시 사용자 부하 테스트.
- 게이트웨이 레이트리밋·타임아웃·재시도·캐시 영속화, HCX↔Qwen 자동 폴백.
- **의존**: 사내 배포 환경/볼트/모니터링 스택 사양. → 운영 인프라 확정 후 진행.

### 1. 인증·접근제어(SSO) [보류]
- 사내 SSO(OAuth/SAML/사번) 연동, FastAPI 인증 미들웨어, 부서·역할별 권한.
- 프론트 로그인 플로우. 모든 `/ask` 에 사용자 신원 주입.
- **의존**: 사내 SSO 프로토콜·엔드포인트 사양. → 사양 확인 후 진행.
- **연계**: 인증이 생기면 로깅(2)의 user_id 가 실제 신원으로 채워짐(현재는 'anonymous').

### 6. 보안·컴플라이언스 [보류]
- 상담 로그 PII 마스킹·보존정책, 접근 감사, 사내 보안 검토.
- 키 관리 볼트화. 답변 출처 추적성(이미 구현)으로 감사 대응.
- **의존**: 사내 보안/개인정보 정책. → 정책 확인 후 진행.

## 이번 사이클 구현 (2·3·5)

### 2. 질의·답변 로깅 + 통계 대시보드
- 모든 질의/답변/출처/라우팅·모드·신뢰도/지연시간을 `data/query_log.db` 에 적재.
- 👍/👎 피드백 수집(`POST /feedback`).
- 집계 통계(`GET /stats`): 총 질의수, 모드 분포, '규정에 없음' 반환율, 평균 지연, 피드백 집계, 빈출 질의.
- 간단 대시보드(HTML)로 시각화. (user_id 는 SSO(1) 전까지 'anonymous')

### 3. 월별 자동 재인덱싱 스케줄러
- `scripts/scheduled_reindex.py`: 재추출→재임베딩/적재→**회귀 게이트(eval) 통과 시에만 버전 반영**.
- 재인덱싱 이력을 `data/reindex_versions`(jsonl)에 기록(시각·산출물 수·게이트 결과·소스 해시).
- 유효일자 버전 관리: 운영기준 행은 `effective_date` 보존, 조회는 최신본 우선.
- 사내 cron/잡 스케줄러 등록 가이드 제공(실행 자체는 사내망).

### 5. CI 회귀 게이트 + A/B 테스트 하네스
- `scripts/ci_gate.py`: pytest + eval 실행, 임계(수치정확도 100%, 정확도≥95%, 출처율 100%) 미달 시 실패.
- `eval/ab_test.py`: 골든셋 결과를 **베이스라인 스냅샷(eval/baseline.json)** 과 비교 →
  항목별 개선/회귀/동일, 지표 델타 리포트. 업그레이드 때마다 A/B 로 회귀를 잡는다.
- 모든 테스트 코드는 `tests/`·`eval/` 에 파일로 보존(업그레이드 시 재실행).
