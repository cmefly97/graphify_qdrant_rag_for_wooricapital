# PRD — 무엇을 만드는가

## 1. 문제와 사용자
- 사용자: 우리캐피탈 오토운영팀 **상담사**(지점 문의 응대).
- 문제: 심사·운영 기준이 FAQ/규정/월별 운영기준 3종에 흩어져 있고, 금리·등급 등 수치가 자주 바뀌어 정확한 답을 빠르게 찾기 어렵다.
- 해결: 자연어 질문 → 근거 기반 정답 + 출처를 즉시 제공.

## 2. 데이터 출처
| 출처 | 내용 | 형식 |
|---|---|---|
| FAQ | 지점 일반 문의 Q-A (52행) | xlsx (일부 답변 "이미지 참조") |
| 규정 | 오토운영팀 핵심 내부규정 | docx, md |
| 운영기준 | 월별 금리·운영기준 | pdf, docx |
파싱 대상: docx, pdf, md, xlsx, 이미지.

## 3. 핵심 기능 (MVP)
1. 자연어 질의 응답(채팅).
2. 하이브리드 검색: 수치형 질의는 테이블 정확조회, 설명형은 그래프RAG.
3. 쿼리 확장(동의어·영문·유사표현)으로 예시 밖 질문 대응.
4. 근거 출처 표시(문서·조항·유효일자) + 근거 그래프 시각화.
5. 예시 질문 칩 제공.
6. 평가 골든셋 기반 정확도 측정.

## 4. 대표 사용자 스토리 & 수용 기준
| 질의 | 기대 답변 | 경로 | 수용 기준 |
|---|---|---|---|
| 론/할부 취급 가능 개월수? | 12~72개월 | 수치형/테이블 | 정확값, 출처 표기 |
| NICE 885점·금리등급 2등급 최저금리? | 21.0% G/L, NICE 1%네고, 거점장11.0/증빙15.0/HJ18.0 | 수치형/테이블 | 수치 100% 일치 |
| 듀얼상품 금리등급 몇등급까지? | Dual_C/O 내·외국인 7등급 | 수치형/테이블 | 정확값 |
| 엔카 슬라이딩 가능해? | 조건부 가능(연식·주행거리·카히스토리…) | 혼합형 | 조건 누락 없음 |
| 신용회복/개인회생 R판정? | 필터링 취급 불가 | 설명형/그래프 | 정확 판정 |

## 5. 비기능 요구
- 정확도: 수치형 답변 오답률 0% 목표(불확실 시 "확인 필요" 표기).
- 신뢰: 모든 답변에 출처. 근거 없으면 "명시되어 있지 않음".
- 응답: 스트리밍, 체감 수 초 내 시작.
- 보안: 키는 .env, 사내 게이트웨이만 사용.
- 운영확장성: 인증·로깅·재인덱싱을 후속에 끼울 수 있는 구조.

## 6. 범위 밖 (후속)
사내 SSO, 접근제어, 질의로그 대시보드, 자동 재인덱싱 스케줄러.

## 7. 성공 지표
- 골든셋(50~100문항) 수치 정확도·출처 정확도·환각률.
- 상담사 체감: 예시 밖 변형 질문 응답 성공률.
