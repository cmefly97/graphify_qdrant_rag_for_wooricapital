# UI_GUIDE — 어떻게 보여줄 것인가 (선택)

## 1. 화면 구성
- **상단**: 서비스명 "우리캐피탈 오토운영팀 상담챗봇" + 예시 질문 칩(4~6개).
- **중앙**: 채팅 스트림(질문/답변 말풍선, 답변은 SSE 스트리밍).
- **답변 카드**: 본문 + "근거" 접이식 섹션(문서명·조항·유효일자, 클릭 시 원문 청크 펼침).
- **우측/하단(접이식)**: 근거 그래프 패널 — 이번 답변이 거친 노드·관계 시각화.

## 2. 예시 질문 칩 (초기)
- 론/할부 취급 가능 개월수?
- NICE 885점·금리등급 2등급 최저금리?
- 듀얼상품 금리등급 몇등급까지?
- 엔카 슬라이딩 가능해?
- 신용회복/개인회생 R판정?

## 3. 답변 표현 규칙
- **수치형 답변**: 표 형태로 구조화 표시(금리등급/GL금리/네고 항목별).
- **출처 배지**: 각 답변에 `[문서명 · 유효일자]` 배지. 클릭 시 원문.
- **불확실/없음**: 회색 안내문 "규정에 명시되어 있지 않습니다" + 관련 규정 위치 링크.
- **충돌**: 최신본 우선 표시 + "과거 기준도 존재" 토글.

## 4. 근거 그래프 패널
- Cytoscape.js. 시작노드(벡터매칭) 강조, 확장노드/관계 라벨(references, similar_to) 표시.
- 노드 클릭 → 해당 원문 청크로 점프.

## 5. 상태/피드백
- 로딩: "검색 중 → 근거 정리 중 → 답변 생성 중" 단계 표시.
- 답변 하단 👍/👎 → 평가 로그(후속 대시보드 연동 자리).

## 6. 접근성/톤
- 한국어 우선, 도메인 용어 그대로. 명료·간결. 과한 장식 지양.
