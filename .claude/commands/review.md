# /review — 규칙 기반 리뷰

현재 변경사항(또는 지정 경로 `$ARGUMENTS`)을 아래 체크리스트로 리뷰하고, 위반 항목을 **심각도(블로커/경고/제안)**로 분류해 보고한다. 추측성 칭찬은 금지, 근거(파일·라인)를 제시한다.

## A. 불변 규칙 위반 (블로커)
- [ ] 수치(금리·등급·금액·개월·판정)를 테이블 조회가 아닌 LLM 자유생성으로 만드는 코드가 있는가?
- [ ] 출처 메타 없이 답변을 반환하는 경로가 있는가?
- [ ] 근거 없을 때 "모름" 대신 추측 답변하는가?
- [ ] 외부(비-사내) LLM/임베딩 호출이 있는가?
- [ ] API 키/시크릿이 코드·로그·테스트·커밋에 하드코딩되었는가? `.env`가 `.gitignore`에 있는가?

## B. 아키텍처 정합성 (경고)
- [ ] 모듈 경계(router/expander/retriever/reranker/generator/tables…)가 ARCHITECTURE.md와 일치하는가?
- [ ] 게이트웨이 호출에 타임아웃·재시도·캐시가 있는가?
- [ ] 유효일자 충돌 시 최신본 우선 로직이 있는가?

## C. 품질 (경고/제안)
- [ ] 핵심 로직에 단위테스트가 있는가? 통과하는가?
- [ ] 타입힌트/ruff/black 통과?
- [ ] 에러 처리·로깅이 충분한가?

## D. Phase Exit Criteria
- [ ] 현재 `phases/phase_<N>.md`의 Exit Criteria 각 항목을 충족하는가? 미충족 항목 명시.

## 출력 형식
1) 블로커 목록(파일:라인 + 근거 + 수정안)
2) 경고 목록
3) 제안 목록
4) 종합 판정: PASS / FIX-REQUIRED
